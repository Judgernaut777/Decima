## Install-from-artifact verification

The wheel was installed with `pip install --no-deps --no-index --find-links <build> --target
<throwaway>` into a directory **outside the repository**, then imported from an unrelated cwd
(`/tmp`) so the source tree cannot shadow it:

- `decima.__version__` → `0.3.0`
- `decima.shell.serve.FRONTEND_DIR` resolves **inside the install target** (not the repo)
- **18** frontend files ship, including `index.html`, `js/screens/qa.js`, `js/screens/workspace.js`,
  and `js/screens/plans.js` (the three Path-A screens)
- `GET /` → **200** with a strict `Content-Security-Policy` header
- unauthenticated `GET /api/v1/tasks` → **401**

Digests are stable for the release commit; rebuild reproducibility depends only on the pinned
source tree (the wheel is `py3-none-any`, platform-independent).
