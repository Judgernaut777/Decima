## decima.models — model routing (Phase 6)

Models are **proposal engines with zero authority** (invariant 4: models propose, deterministic code authorizes). Nothing in this package mints, holds, or checks authority; turning a proposal into an effect requires the kernel's authorization + approval + receipt chain, which lives **outside** this package (and which this package does not import — enforced by a test).

### Modules
- **`providers.py`** — the structural `ModelProvider` Protocol (`capabilities()`/`complete()`/`stream()`); `DeterministicProvider` (rule-based, offline, reproducible — output is derived from the kernel content hash of the request, never a clock/random; the default fallback and the engine every test uses); thin `LocalProvider`/`CloudProvider` adapters that conform structurally but make **no live network call** — a live call is gated behind an injected `backend` seam and they **fail closed** (`LiveTransportRequired`) without one. Secrets are applied by a broker at call time (`broker.use_secret`), never stored on the object, embedded in code/context, or logged. `ModelResponse` and `ProposedAction` are inert DATA (`instruction_eligible=False`, no effect method).
- **`registry.py`** — `ModelRegistry`/`ModelEntry`: the catalogue (provider, local/remote, context limit, modalities, structured-output, tool-use, est cost in micro-cents, privacy class, enabled). Pure config; binding a provider grants nothing.
- **`routing.py`** — `RoutingPolicy.select(TaskSpec, registry) → RoutingDecision(selected_model, fallback_models, reason_codes, estimated_cost, context_policy)`. Hard eligibility filters (sensitive ⇒ local-only, modality/context/structured/tool/cost) then a deterministic integer ranking (sensitive/realtime prefer local, then cheapest, ties by id). The decision is DATA; `record(k, decision)` folds it onto the Weft as a `model_routing` Cell. `route_and_complete(...)` drives the decision through providers and, on failure/refusal, falls through the recorded fallback chain **bounded by `max_hops`**.
- **`budgets.py` / `accounting.py`** — `UsageLedger`/`UsageRecord` (token + cost accounting, all ints, caller-supplied latency so nothing reads a clock) and `BudgetGuard` with a pre-check (`check`) that denies and a `spend` that **stops** (raises `BudgetExceeded` without running the call) once a token/cost cap is reached.
- **`validation.py`** — structured proposals validated against explicit declarative schemas; invalid proposals are **rejected** (never eval-repaired) and recordable as `model_error` Cells, with a bounded `validate_with_reprompt` path that executes nothing when exhausted.

### Invariants honored
Zero ambient authority (3); models propose / code authorizes (4); untrusted content is data (5, `instruction_eligible=False`); ints-not-floats and no wall-clock/random in recorded content (6). The Weft stays the sole canonical store — routing/usage/error records are provenance DATA folded via `decima.kernel.model.assert_content`.
