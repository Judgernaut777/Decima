## Roadmap ‚Äî the path to a working version (planned)

The cycle log below is *what's done*. The forward plan ‚Äî consolidated as the source of truth in
[`VISION.md`](../VISION.md) ‚Üí **"The road to a working version"** ‚Äî is six phases, and **the ordering
is a gate: enforcement must be real before anything goes live.**

1. **Enforcement** ‚Äî untrusted-content quarantine boundary ¬∑ real worker isolation (seccomp/landlock/microVM/WASM) ¬∑ network egress boundary ¬∑ sync channel confidentiality + peer auth. *(Blocks Phase 2.)*
2. **Go live** ‚Äî model brain as the default driver ¬∑ a few engines live against real accounts ¬∑ a real surface with an approval inbox. *(First daily-driver version.)*
3. **Self-extension** ‚Äî the forge-real loop: intent ‚Üí codegen ‚Üí sandboxed test ‚Üí scan ‚Üí attested promotion ‚Üí versioning.
4. **Always-on substrate** ‚Äî durable scheduling/background across restart ¬∑ crash-resumable execution ¬∑ concurrency ¬∑ observability + live spend metering ¬∑ live-world test harness ¬∑ key rotation/recovery ¬∑ schema migration ¬∑ backup/restore.
5. **Full surface, citizens, mediated I/O, knowledge** ‚Äî accreting voice-first Shell ¬∑ terminals-as-citizens + real MCP mount/expose ¬∑ sandboxed email digest + mediated browser ¬∑ personal-corpus ingestion ¬∑ multi-human ¬∑ install/self-update.
6. **The single Rust port** ‚Äî last, gated on the reference being stable with this roadmap green.
